# Agent Index — File Formats

This document specifies the JSON schemas for the **layer-2 deterministic
indices** generated by `scripts/build-symbols.sh` and `scripts/build-logs.py`,
and consumed by the `context-discovery` sub-agent.

These files live at:

- `.agent-index/symbols.json`
- `.agent-index/logs.json`

Both are committed to the target repo and refreshed on every push by the
`Build agent index` workflow. They are **machine-readable, deterministic** —
not LLM output.

---

## Conventions

- Top-level fields:
  - `schema` — string identifier, e.g. `agent-index/symbols`
  - `version` — integer, bumped on backward-incompatible changes
  - `generated_at` — UTC ISO 8601 timestamp
  - `generator` — string identifying the producing tool + version
  - `repo.commit` — git SHA the index was built from (or `"unknown"`)
- File paths use forward slashes, relative to repository root.
- Arrays are sorted (`file`, then `line`) for deterministic diffs.
- All optional string fields use `null` when absent (not `""`).

## `agent-index/symbols` (v1)

Produced by `scripts/build-symbols.sh`, which wraps `universal-ctags`.

### Schema

```json
{
  "schema": "agent-index/symbols",
  "version": 1,
  "generated_at": "2026-06-25T10:00:00Z",
  "generator": "Universal Ctags 5.9.20240414(...)",
  "repo": { "commit": "<sha>" },
  "symbols": [
    {
      "name": "ParseFlowID",
      "kind": "function",
      "language": "Go",
      "file": "pkg/flowtopic/flowtopic.go",
      "line": 42,
      "signature": "(s string) (FlowID, error)",
      "scope": null,
      "exported": true
    }
  ]
}
```

### Field reference

| Field | Type | Notes |
|---|---|---|
| `name` | string | Symbol identifier as it appears in source. |
| `kind` | string | One of: `function`, `method`, `class`, `struct`, `interface`, `type`, `const`, `var`, `field`, `enum`, `package`, `module`, `unknown`. Values come from ctags; not normalized across languages. |
| `language` | string | ctags-reported language: `Go`, `Java`, `Python`, `JavaScript`, `TypeScript`, `Rust`, `Ruby`, `C`, `C++`, `unknown`. |
| `file` | string | Path relative to repo root. |
| `line` | integer | 1-based line number of definition. |
| `signature` | string \| null | Function/method signature where available; `null` for types/vars. |
| `scope` | string \| null | Containing scope (class/package) when ctags reports one. |
| `exported` | boolean | True if `name` starts with an uppercase letter. Convenient for Go visibility checks; treat as a heuristic for other languages. |

### Exclusions

- `vendor/`, `node_modules/`, `.git/`, `dist/`, `build/`, `target/`, `__pycache__/`, `.venv/`, `venv/`, `.agent-index/`
- Generated protobuf code: `*.pb.go`, `*.pb.validate.go`, `*_pb2.py`, `*_pb2_grpc.py`
- Bundled JavaScript: `*.min.js`, `*.bundle.js`

### Caveats

- **Symbol kinds are language-specific.** Don't compare `kind` across languages without a mapping table.
- **Macros / generated code are skipped** if they live in excluded paths; if they don't, they appear like any other symbol.
- **Duplicates by (name, file, line) are deduplicated.** Same name in different files/lines is kept.

## `agent-index/logs` (v1)

Produced by `scripts/build-logs.py`, which scans source files with per-language
regex for logger call sites.

### Schema

```json
{
  "schema": "agent-index/logs",
  "version": 1,
  "generated_at": "2026-06-25T10:00:00Z",
  "generator": "build-logs.py v1",
  "repo": { "commit": "<sha>" },
  "languages": ["Go", "Java", "Python"],
  "logs": [
    {
      "level": "error",
      "message": "failed to consume from kafka: %v",
      "literal": true,
      "file": "pkg/kafkaconsumer/consumer.go",
      "line": 118,
      "language": "Go"
    },
    {
      "level": "info",
      "message": "",
      "literal": false,
      "file": "pkg/svc/zserver.go",
      "line": 204,
      "language": "Go"
    }
  ]
}
```

### Field reference

| Field | Type | Notes |
|---|---|---|
| `level` | string | Normalized: `trace`, `debug`, `info`, `warn`, `error`, `fatal`. |
| `message` | string | The literal first argument with surrounding quotes stripped. Empty string when `literal=false`. |
| `literal` | boolean | `true` if the first argument was a string literal; `false` for variables, concatenations, template strings, or empty call sites. |
| `file` | string | Path relative to repo root. |
| `line` | integer | 1-based line number of the call site. |
| `language` | string | One of: `Go`, `Java`, `Python`, `TypeScript`. |

### Level normalization map

| Source token | Normalized |
|---|---|
| `trace` | `trace` |
| `debug` | `debug` |
| `info`, `log`, `print` | `info` |
| `warn`, `warning` | `warn` |
| `error`, `exception` | `error` |
| `fatal`, `critical`, `panic` | `fatal` |

### Recognized logger receivers

The regex matches identifiers that look like loggers. The exact list per
language:

- **Go**: `log`, `logger`, `zlog`, `zlogger`, `slog`, `logr`, `klog`, `sugar`, `l`,
  with optional struct-field traversal and capitalized method chain.
  Methods: `Info|Warn|Warning|Error|Debug|Fatal|Panic|Print|Trace` with optional
  `f`/`w`/`S`/`ln` suffix.
- **Java**: `log`, `logger`, `LOG`, `LOGGER`. Methods: `info|warn|warning|error|debug|trace|fatal`.
- **Python**: `log`, `logger`, `logging`, `_log`, `_logger`. Methods:
  `info|warning|warn|error|debug|critical|exception`.
- **TypeScript / JavaScript**: `console`, `log`, `logger`. Methods:
  `log|info|warn|warning|error|debug|trace`.

### Limitations (intentional, v1)

- **Only the first argument is captured.** Bindings / format args are not extracted.
- **Only literal first args produce a `message`.** Variables, concatenations,
  and template strings yield `literal=false, message=""`. The file:line is
  still recorded — agents can read the line directly if needed.
- **Single-line patterns only.** Multi-line log statements partially captured.
- **Comment-leading lines are skipped** by checking the first non-whitespace
  characters. End-of-line comments containing log-like syntax are NOT filtered.
- **No call-tree analysis.** A function `wrapLog(msg)` that internally calls
  `log.Info(msg)` is captured at the `log.Info` site, not the caller.

### Caveats

- Logger libraries that wrap a custom struct method (e.g. `s.log("...")`) will
  not match unless the receiver identifier matches the recognized list. Add
  new receivers by editing `LANG_PATTERNS` in `scripts/build-logs.py` and
  re-tagging the templates repo.
- The output is best-effort. For incident triage it's far better than
  scanning the codebase from scratch; for compliance / audit it is not
  authoritative.

## Bumping a schema version

If you change a field's meaning or remove a field, bump `version`. Add a
`CHANGELOG.md` entry alongside this doc and update any consumer agents to
handle both versions until producers have caught up.

Backward-compatible additions (new optional fields) do **not** bump `version`.
